> **Legacy.** `Modal` is kept for backward compatibility. For new code,
> prefer [`Dialog`](/docs/components/overlays/dialog) — it exposes the
> same behavior (portal, focus trap, scroll lock, focus restore) plus
> composable `Dialog.Header` / `Dialog.Body` / `Dialog.Footer` slots and
> `default | danger | warning | info` variants. For simple confirm / cancel
> prompts, use [`ConfirmDialog`](/docs/components/overlays/confirm-dialog).

## Installation

```tsx
import { Modal } from '@camtomlabs/malix-design-system';
```

## Usage

```tsx
const [open, setOpen] = useState(false);

<Modal
  open={open}
  title="Confirm Action"
  onClose={() => setOpen(false)}
  onConfirm={() => {
    handleConfirm();
    setOpen(false);
  }}
>
  <p>Are you sure you want to proceed? This action cannot be undone.</p>
</Modal>
```

## With Custom Labels

Override the default button labels to match your context:

```tsx
<Modal
  open={open}
  title="Delete Project"
  onClose={() => setOpen(false)}
  onConfirm={handleDelete}
  confirmLabel="Delete"
  cancelLabel="Keep"
>
  <p>This will permanently delete the project and all its data.</p>
</Modal>
```
