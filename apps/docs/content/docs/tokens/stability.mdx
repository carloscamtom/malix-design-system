# Token Stability Contract

Every CSS custom property in the Malix Design System is classified as **Stable** or **Experimental**. This contract tells consuming teams which tokens are safe to depend on and which may change.

## Deprecation Policy

| Classification | Semver guarantee | What it means |
|---|---|---|
| **Stable** | No breaking changes without a **major** version bump | Safe to use in production. Will not be renamed, removed, or semantically changed in minor/patch releases. |
| **Experimental** | May change in **minor** releases | Under active evaluation. API (name, default value, or semantic meaning) may change. Use at your own risk. |

### How deprecation works

1. A token is marked `@deprecated` in the registry and release notes
2. It continues to work for at least **one minor release cycle**
3. It is removed in the next **major** release
