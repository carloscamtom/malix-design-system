**Malix** is the design system behind [Camtom](https://camtomx.com), a B2B SaaS platform that helps customs agencies and import/export teams work faster using AI — from document extraction to HS classification and tariff calculation.

When you're building compliance-ready enterprise software, visual consistency isn't a nice-to-have. It's a requirement. Every misaligned input, every inconsistent status indicator, every off-brand screen erodes the trust that customs professionals need to do their job. Malix exists to make that trust automatic.

## Why "Malix"?

In Yucatán slang, a *malix* is a stray dog of unknown origin that becomes domestic over time — scrappy, adaptable, and eventually indispensable.

That's what this design system does. It takes the visual chaos of a growing product — the one-off components, the ad-hoc spacing, the "just this once" color overrides — and domesticates it into a predictable, scalable UI language.

## Design Principles

### Precision Over Decoration

Enterprise compliance software demands clarity. Every pixel serves a purpose: communicating status, guiding workflows, and reducing cognitive load for professionals who process hundreds of shipments per day.

### Token-Driven

All visual properties come from a closed set of 74 CSS custom properties (`--malix-*`). No raw hex colors, arbitrary pixel values, or magic numbers. Every token is validated at build time.

### Data-Attribute Variants

Component variants are controlled via `data-*` attributes, not CSS modifier classes. This keeps the API clean and the styles predictable:

```css
.malix-button[data-hierarchy="primary"] { ... }
.malix-card[data-level="2"] { ... }
```

### No Surprise Policy

New tokens and components require proposal and approval before implementation. The system grows intentionally, not accidentally — because in regulated industries, surprises are expensive.

## What's Inside

- **49 Components** — From simple atoms like Divider and Avatar to complex patterns like DataTable, SidebarPanel, and PricingCard.
- **74 Design Tokens** — A closed, validated set of CSS custom properties for colors, spacing, typography, and shadows.
- **Zero Runtime Dependencies** — Components have no external dependencies beyond React.
- **TypeScript First** — Full type safety with exported interfaces for every component.
- **Accessible** — Keyboard navigation, focus management, ARIA attributes, and screen reader support built in.

## Built at Camtom

Malix wasn't designed in a vacuum. Every component was born from a real product need — a table that had to handle 500-row tariff lookups, a status indicator that had to communicate five classification states at a glance, a sidebar that had to scale across three product verticals.

If you're building enterprise tools where consistency and trust matter, Malix gives you a head start.
